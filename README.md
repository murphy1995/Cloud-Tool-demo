# 云存储迁移调研与评估工具（Cloud-Tool-demo）

**简体中文** | [English](README.en.md)

一个纯前端的单页应用，用于在售前调研 / 方案评估阶段快速估算云存储迁移的**时长、有效吞吐、迁移架构与优化建议**。输入客户环境参数后，页面实时生成评估指标、Mermaid 架构图、时长分解条形图与针对性建议。

## 功能特性

- **实时评估**：任意参数变化即触发重算（带 120ms 防抖），无需点击提交。
- **多介质模型**：内置块存储、文件存储、对象存储三套迁移参数模型（协议、迁移工具、元数据开销、传输效率、小文件惩罚系数）。
- **迁移时长估算**：将总时长拆分为「环境准备 → 数据传输 → 元数据同步 → 校验收尾」四个阶段，并给出含 20% 缓冲的预估总时长。
- **跨地域因子**：根据源 / 目标位置自动套用同区、同国、跨境、本地到云的延迟系数。
- **小文件惩罚**：平均文件小于 1 MB 时自动放大元数据同步耗时，并触发小文件优化建议。
- **并行加速**：并行通道数按 √N（上限 4 倍）影响元数据吞吐。
- **可视化架构图**：基于 Mermaid 动态生成「源 → 迁移网关 → 网络 → 目标」流程图。
- **智能建议**：依据存储类型、文件大小、跨境距离、带宽占比生成迁移注意事项与优化方向。
- **中英双语**：右上角按钮一键切换中文 / English，自动记忆上次选择并按浏览器语言初始化。

## 快速开始

项目无构建步骤、无依赖安装。Mermaid 通过 CDN 引入，**运行需联网**。

直接用浏览器打开 `index.html` 即可。若浏览器对本地文件加载脚本有限制，可启动一个静态服务器：

```powershell
# 任选其一
python -m http.server 8080
# 或
npx serve .
```

然后访问 `http://localhost:8080`。

## 输入参数说明

| 参数 | 说明 |
| --- | --- |
| 数据量 | 总数据规模，支持 GB / TB / PB |
| 文件数量 | 用于计算平均文件大小与元数据开销 |
| 存储介质 | 块存储 / 文件存储 / 对象存储，决定迁移模型 |
| 源存储位置 | 含中国多区域、香港、新加坡、东京、法兰克福、弗吉尼亚及本地 IDC |
| 迁移目标 | 同上位置列表 |
| 迁移网络带宽 | 支持 Mbps / Gbps |
| 并行传输通道数 | 1–32，影响元数据与对象迁移吞吐 |

## 评估输出

- **核心指标**：预估总时长（含缓冲）、基准时长、有效吞吐（Mbps 与 TB/小时）、总数据量。
- **迁移架构图**：随存储类型与位置动态渲染的 Mermaid 流程图。
- **时长分解**：四阶段耗时条形图。
- **架构与迁移建议**：推荐链路、小文件优化、跨境传输、各介质注意事项、带宽瓶颈提示。

## 估算模型概要

```
有效带宽   = 带宽 × 传输效率 × (1 / 跨地域因子)
传输耗时   = 数据量(bit) / 有效带宽
元数据耗时 = 文件数 × 单文件元数据耗时 × 小文件系数 / 并行加速(√N, ≤4)
准备耗时   = 块存储 2h / 文件存储 1h / 对象存储 0.5h
校验耗时   = 传输耗时 × 8% + 文件数 × 0.001s
总时长     = 准备 + 传输 + 元数据 + 校验
预估总时长 = 总时长 × 1.2（20% 缓冲）
```

> 估算基于典型云迁移场景模型，实际时长受网络抖动、小文件比例、API 限流等因素影响，建议预留 15%–30% 缓冲。
